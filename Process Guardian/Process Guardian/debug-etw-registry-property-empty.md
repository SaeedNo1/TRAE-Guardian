# Debug Session: etw-registry-property-empty

## Status: [IN PROGRESS]

### Problem Description
ETW注册表事件捕获后，从事件中提取`KeyName`属性返回空值，导致AI分析无法触发。

### Symptoms
- ETW监控已启用，注册表事件之前能被捕获（20:00:34）
- 最近测试中没有捕获到注册表事件
- `GetPropertyString(ev, L"KeyName", ...)` 返回空字符串
- AI分析从未触发

### Expected Behavior
- 从ETW注册表事件中正确提取注册表路径和值名称
- AI分析模块能根据注册表操作触发风险评估

### Hypotheses
1. **ETW事件属性名称错误**: 内核注册表提供者可能使用不同的属性名称（如`ObjectName`而非`KeyName`）
2. **属性数据类型错误**: 属性可能不是字符串类型，而是二进制或其他类型
3. **ETW事件版本不匹配**: 不同Windows版本的注册表事件格式不同
4. **属性索引位置错误**: 需要按索引而非名称获取属性
5. **ETW会话配置错误**: 可能缺少必要的关键字段或级别设置

### Evidence Collected

**已确认问题：**
- ✅ ETW监控启动成功，所有提供者都已启用（status=0）
- ✅ 之前测试中注册表事件能被捕获（20:00:34）
- ✅ **发现bug**: 事件ID 21被错误地当作进程启动事件处理，但实际上是镜像加载事件，导致同一PID被反复处理，最终导致守护进程崩溃
- ✅ **已修复**: 移除了事件ID 21作为进程启动事件的处理

**待确认问题：**
- ⚠️ 最近测试中没有捕获到注册表事件，需要进一步调试
- ⚠️ `DumpAllProperties`函数已添加，但由于守护进程崩溃，未收集到属性信息

### Fix Applied
1. **修复ETW事件过滤逻辑**: 在`EventRecordCallback`中移除了`(id == 21 && opcode == 0)`作为进程启动事件的处理

### Verification Result
- ✅ 守护进程运行时间更长（从几秒到超过1分钟）
- ✅ 性能测试结果良好（2.56秒完成100进程+50文件操作）
- ⚠️ 注册表事件仍未捕获到，需要进一步调试
